1
Check HA cluster status
read-only
get system ha status
diagnose sys ha status
diagnose sys ha dump-by vcluster
💡 Shows primary/secondary role, uptime, sync state, heartbeat link health.
2
Verify synchronization
⚠ config
diagnose sys ha checksum show
diagnose sys ha checksum recalculate
execute ha synchronize start
💡 Checksum mismatch = config divergence. Force sync to resolve.
3
Force manual failover
⚠ impact
execute ha failover set 1
get system ha status
execute ha failover unset 1
💡 ⚠️ Brief traffic interruption expected. Confirm new primary role after.

AI Diagnosis — LLaMA 3.3 70B
Root cause: Phase 1 IKE proposal mismatch. The error no proposal chosen means the initiator is advertising encryption/DH-group combinations the remote peer cannot accept.

Fix commands:
show vpn ipsec phase1-interface SiteA-HQdiagnose debug application ike -1diagnose debug enable
Resolution: Align set proposal aes256-sha256 and set dhgrp 14 on both peers. Switch to IKEv2.
